## 3. Padrões de Código

### TypeScript
- Use TypeScript para **tudo**
- Não use `any` sem justification
- Exporte tipos e interfaces de `types/`

### SolidJS
- Use `createStore` para estados de jogo complexos
- Use componentes funcionais com reatividade granular
- Mantenha lógica de Socket **separada** da UI
- Use `<Show>`, `<For>`, `<Switch>` para controle de fluxo

### Socket.io
- Eventos em **snake_case**: `CHOOSE_WORD`, `START_WRITING`, `GAME_STATUS_CHANGED`
- Mantenha event handlers em arquivos separados (`handlers/`)
- Registre eventos em `events/socket.events.ts`

### Anti-Cheat
- **NUNCA** envie `meaning` (definição) para os clientes
- Payload `START_WRITING` contém apenas: `{ word: string, leaderName: string }`

### Logging
- Use prefixos para categorizar logs:
  - `[Game]` - Eventos de jogo
  - `[PlayersStore]` - Gerenciamento de jogadores
  - `[Words]` - Operações com palavras
  - `[Conexão]` - Conexões socket

## 4. Eventos de Socket

### Server → Client
| Evento | Payload | Destino |
|--------|---------|---------|
| `START_WRITING` | `{ word, leaderName }` | Apenas jogadores (não líder) |
| `LEADER_CONFIRMED` | `{ status, message }` | Apenas líder |
| `GAME_STATUS_CHANGED` | `{ status, leaderName }` | Todos |
| `playersList` | `Player[]` | Todos |

### Client → Server
| Evento | Payload |
|--------|---------|
| `CHOOSE_WORD` | `{ wordId, wordText }` |
| `identify` | `{ profile, playerName }` |
| `startGame` | `{}` |

## 7. Fluxo do Jogo

1. **Lobby** - Host inicia, QR Code gerado
2. **Leader Selection** - Líder escolhe palavra
3. **Writing** - Jogadores enviam definições (pizza)
4. **Voting** - Jogadores votam na melhor definição
5. **Results** - Pontuação e próxima rodada
