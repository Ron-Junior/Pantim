# Regras Globais do Projeto - Pantim (Fictionary LAN)

Você é um desenvolvedor expert em Node.js e SolidJS. Sempre que eu pedir algo, siga estas diretrizes AUTOMATICAMENTE:

## 1. Stack Tecnológica
- Backend: Node.js (TypeScript) + Socket.io.
- Frontend: SolidJS + Tailwind CSS.
- Build: Vite.

## 2. Padrões de Código
- Use TypeScript para tudo.
- No SolidJS, prefira `createStore` para estados de jogo complexos.
- Use componentes funcionais e reatividade granular do Solid.
- Mantenha a lógica de Sockets separada da lógica de UI.

## 3. Comportamento Esperado
- Ao criar rotas, lembre-se que o projeto roda em LAN (use IPs locais, não localhost).
- Sempre considere que existem dois perfis: "Host" (Tela da TV) e "Player" (Celular).
- As definições de tipos devem ser compartilhadas entre server e client na pasta `/shared`.

## 4. Contexto de Negócio
- Este é um clone de Fictionary/Jackbox.
- O foco é diversão, humor e rapidez na rede local.
