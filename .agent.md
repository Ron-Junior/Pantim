### Socket Events
**Incoming (Client в†’ Server):**
- `identify` - Player/Host identification
- `CHOOSE_WORD` - Leader selects word { wordId, wordText }
- `startGame` - Host starts game
- `startRound` - Start round with word

**Outgoing (Server в†’ Client):**
- `START_WRITING` - { word, leaderName } (sent to players, not leader)
- `LEADER_CONFIRMED` - Confirmation to leader
- `GAME_STATUS_CHANGED` - Status updates
- `playersList` - Connected players

## Code Patterns

### Server: Adding New Event Handler
1. Create handler in `handlers/`
2. Export function that receives `(io, socket, data)`
3. Register in `events/socket.events.ts`

### Server: Game State Management
```typescript
import { updateGameStatus, getGameState } from '../services';

updateGameStatus(GameStatus.WRITING);
const state = getGameState();
```

### Client: Socket Usage
```typescript
import { socket } from '../lib/socket';

socket.on('START_WRITING', (data) => {
  setStore('currentWord', data.word);
});
```

### Client: Store Pattern
```typescript
import { createStore } from 'solid-js/store';

const [gameStore, setGameStore] = createStore({
  status: 'WAITING',
  word: '',
  leaderName: ''
});
```

## When Assisting With This Project
- Always use TypeScript strict mode
- Follow the server directory structure (types/services/handlers/events)
- Keep socket logic separate from UI components
- Consider both Host and Player code paths
- Use shared types between server/client via types file
- Log significant game events with `[Game]` prefix
- Validate Socket.io event names match exactly
- For anti-cheat: NEVER send word meaning to clients, only word text
